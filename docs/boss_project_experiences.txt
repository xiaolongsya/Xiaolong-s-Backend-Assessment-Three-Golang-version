项目一：OpenAI 规范对话中转服务器（考核三 / Go 版）
担任角色：后端负责人 / 独立开发（Go）
项目描述：
1. 用 Go（Gin + GORM + MySQL）实现“部分 OpenAI API 语义”的后端服务，对外提供统一的模型调用接口，对内可转发到不同上游模型服务。
2. 支持 Chat Completions（SSE 流式与非流式）：stream=true 时返回 text/event-stream，按 chunk 增量输出，并以 data: [DONE] 结束；stream=false 一次性返回标准 JSON。
3. 全接口 Bearer Token 鉴权：通过中间件统一校验 Authorization: Bearer <token>，非法返回 401，并返回 OpenAI 风格错误结构。
4. 生成请求全生命周期：为每次生成创建唯一 completion_id，支持结果查询（GET）、删除（DELETE）与流式任务取消（CANCEL），并将 request/response 持久化到 MySQL。
5. 模型白名单机制：模型列表来源于数据库 ai_models（enabled=1），Chat 请求的 model 必须存在且启用，避免在代码里硬编码模型。
6. 可选扩展：Files API（上传/列表/详情/删除），文件落盘 + 元信息入库，并限制上传大小 20MB，超限返回 413。
7. 可选扩展：上游中转（多 provider / 多 key / 自动回退），按 ai_models.owned_by 选择上游配置；支持同 provider 多 key 分摊；对网络错误、HTTP 5xx、429、401、403 触发回退；并改写上游返回 id 保证生命周期接口一致。
项目业绩（可观测/可核验）：
1. 完成一套可被官方 SDK 直接调用的接口实现：使用官方 OpenAI Python SDK 编写回归脚本，验证 models.list、chat 非流式、chat 流式均可正确解析（具备“兼容性证据”）。
2. 覆盖核心接口链路（可在文档中核验）：healthz、/v1/models、/v1/chat/completions（stream true/false）、/v1/chat/completions/{id}（GET/DELETE/CANCEL）；并提供 OpenAPI（可导入 Apifox）。
3. 将“流式生成”做成真正的服务端流（SSE），而非伪流式：支持逐块输出与取消请求，适配前端/客户端的实时渲染场景。
4. 将生成过程持久化落库，支持后续审计/复查：请求与响应均入库，记录状态（running/completed/failed/cancelled）与取消时间，便于排查与运维。
5. 通过配置化方式接入上游模型服务：使用环境变量管理上游 base_url 与 key，多 provider 与回退链可快速扩展新厂商，降低接入成本。

项目六：校园智能导游软件 CampusLife Compass（团队竞赛 / 私密）
担任角色：后端负责人（Java / Spring Boot）
项目描述：
1. 搭建“校园生活足迹 + 智能规划 + 社区分享”的服务端能力，面向移动端（Uni-app + Vue3）提供 RESTful API。
2. 后端技术栈：Spring Boot + MyBatis-Plus + MySQL（utf8mb4），统一 Result 返回结构；使用 JWT 拦截器实现 Bearer Token 鉴权与用户身份注入。
3. 核心业务模块（可在数据库与 Controller 中核验）：用户体系、地点/POI、足迹打卡（footprint）、路线模板与节点（route_template/route_node）、社区互动（点赞/评论/收藏/克隆）、推送记录、AI 分析报告与规划请求留痕。
4. 足迹模块：支持分页查询与筛选（类型、时间范围），提供创建/更新/删除（软删 status=0）与详情查询接口，支撑“地图足迹 + 时间线 + 数据统计”的上层展示。
5. 地点库模块：实现热门地点表 popular_place 及查询接口（按分类/区域/消费水平/详情），并维护一套“校内+校外”样例数据（文档列出 25 个热门地点），为智能推荐提供候选集。
6. AI 智能服务（纯 Java 实现）：提供用户行为模式分析、活动预测、周报/月报生成与智能路线推荐接口（/ai/analyze、/ai/predict、/ai/report/{type}、/ai/recommend-route），支持时间与预算等约束输入。
7. 接口可视化与联调：启用 SpringDoc/Swagger（白名单放行），便于前后端与团队快速联调与验收。
项目业绩（可观测/可展示）：
1. 设计并落地 12 张核心业务表（user、poi、footprint、shared_footprint_card、route_template、route_node、user_preference、social_interaction、report、push_notification、user_behavior_pattern、route_plan_request），支撑记录-分析-规划-分享的完整闭环。
2. 实现 AIAnalysisService（纯 Java 统计/规则算法）替代 Python 依赖：同一进程内完成分析/预测/报告/路线推荐，降低部署复杂度并提升可维护性（文档与代码均可核验）。
3. 将“推荐”从概念落到数据驱动：地点库支持分类/区域/预算筛选（cost_level），结合用户历史足迹与偏好，输出推荐地点与推荐理由。
